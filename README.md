# Machata Studio Bot

Telegram-бот для бронирования репетиций в Machata studio.

## Функции

- 💰 Показ цен на аренду (700₽/час)
- 📋 Правила студии и условия аренды
- 📅 Бронирование репетиций (в разработке)

## Деплой на Render

### 1. Создать бота в Telegram

1. Откройте [@BotFather](https://t.me/BotFather) в Telegram
2. Отправьте команду `/newbot`
3. Введите имя бота (например, `Machata Studio Bot`)
4. Введите username бота (должен оканчиваться на `bot`, например `machata_studio_bot`)
5. Скопируйте **API Token** (не делитесь им ни с кем!)

### 2. Создать Web Service на Render

1. Перейдите на [Render](https://render.com/) и зарегистрируйтесь
2. Нажмите **New +** → **Web Service**
3. Подключите свой GitHub-аккаунт
4. Выберите репозиторий `machata-studio-bot`
5. Заполните настройки:
   - **Name**: `machata-studio-bot`
   - **Region**: выберите ближайший регион
   - **Branch**: `main`
   - **Runtime**: `Python 3`
   - **Build Command**: `pip install -r requirements.txt`
   - **Start Command**: `python bot.py`

### 3. Добавить переменную окружения

1. В настройках сервиса перейдите в **Environment**
2. Добавьте новую переменную:
   - **Key**: `BOT_TOKEN`
   - **Value**: ваш API Token от BotFather
3. Нажмите **Save Changes**

### 4. Запустить бота

1. Render автоматически запустит деплой
2. Дождитесь статуса **Live**
3. Откройте своего бота в Telegram и нажмите `/start`

## bot_v2.py - Версия с интеграцией YooKassa

Это улучшенная версия бота с полноценной интеграцией платежной системы YooKassa для приема онлайн-оплаты за бронирование студий.

### Особенности bot_v2.py:

- 💳 Интеграция с YooKassa для приема платежей
- 🔔 Уведомления администратора о новых бронированиях
- ✅ Автоматическое подтверждение брони после оплаты
- 📊 Просмотр статуса своих бронирований
- 🎨 Улучшенный интерфейс с эмодзи

### Настройка YooKassa

#### 1. Создайте аккаунт в YooKassa

1. Перейдите на [yookassa.ru](https://yookassa.ru/) и зарегистрируйтесь
2. Пройдите верификацию (потребуются документы компании или ИП)
3. После одобрения перейдите в личный кабинет

#### 2. Получите API ключи

1. В личном кабинете YooKassa откройте раздел **Настройки** → **API и Webhooks**
2. Скопируйте **shopId** (идентификатор магазина)
3. Создайте и скопируйте **Секретный ключ**

#### 3. Настройте переменные окружения в Render

Добавьте следующие переменные в настройках вашего Web Service:

- **YOOKASSA_SHOP_ID**: ваш shopId из YooKassa
- **YOOKASSA_SECRET_KEY**: ваш секретный ключ из YooKassa
- **ADMIN_CHAT_ID**: ваш Telegram Chat ID для получения уведомлений

#### 4. Получите свой Telegram Chat ID

1. Откройте бота [@userinfobot](https://t.me/userinfobot) в Telegram
2. Отправьте ему команду `/start`
3. Он отправит вам ваш Chat ID (например: `123456789`)
4. Используйте этот ID для переменной ADMIN_CHAT_ID

### Запуск bot_v2.py на Render

1. В настройках Web Service измените **Start Command** на:
   ```
   python bot_v2.py
   ```

2. Убедитесь, что все переменные окружения установлены:
   - BOT_TOKEN
   - YOOKASSA_SHOP_ID
   - YOOKASSA_SECRET_KEY
   - ADMIN_CHAT_ID

3. Нажмите **Manual Deploy** для перезапуска сервиса

### Тестирование платежей

YooKassa предоставляет тестовую среду для проверки платежей:

1. Используйте тестовые API ключи из раздела **Тестирование**
2. Для тестовых платежей используйте карту: `5555 5555 5555 4444`
3. Любая дата и CVV

### Настройка цен

Цены на студии настраиваются в файле `bot_v2.py` в разделе:

```python
PRICES = {
    'studio_a': {'name': 'Студия A', 'hourly_rate': 1500, ...},
    'studio_b': {'name': 'Студия B', 'hourly_rate': 1200, ...},
    'studio_c': {'name': 'Студия C', 'hourly_rate': 800, ...}
}
```

### Важные замечания

⚠️ **Безопасность:**
- Никогда не публикуйте секретные ключи в коде
- Всегда используйте переменные окружения
- Регулярно обновляйте API ключи

⚠️ **Webhook (дополнительно):**
Для автоматического обновления статуса платежей настройте webhook в YooKassa:
