# Machata Studio Bot

Telegram-бот для бронирования репетиций в Machata studio с полным функционалом.

## 🎯 Функции

- 📅 **Полноценный календарь** для выбора даты с навигацией по месяцам
- ⏱ **Выбор продолжительности** репетиции (1, 2 или 3 часа)
- 🕐 **Выбор времени** с проверкой занятых слотов
- 📊 **Просмотр своих бронирований** - список всех ваших броней
- 💳 **QR-код для оплаты** - автоматическая генерация QR-кода с реквизитами
- 🔔 **Уведомления администратора** о новых бронированиях
- 📍 **Поддержка нескольких студий** (легко добавить новые)
- 💰 **Информация о ценах и правилах** студии

## 🚀 Быстрый старт

### 1. Создать бота в Telegram

1. Откройте [@BotFather](https://t.me/BotFather) в Telegram
2. Отправьте команду `/newbot`
3. Введите имя бота (например, `Machata Studio Bot`)
4. Введите username бота (должен оканчиваться на `bot`, например `machata_studio_bot`)
5. Скопируйте **API Token** (не делитесь им ни с кем!)

### 2. Установка зависимостей

```bash
pip install -r requirements.txt
```

### 3. Настройка переменных окружения

Создайте файл `.env` или установите переменные окружения:

```bash
BOT_TOKEN=ваш_токен_от_BotFather
ADMIN_CHAT_ID=ваш_telegram_id  # Опционально, для уведомлений
PAYMENT_CARD=номер_карты_или_кошелька
PAYMENT_PHONE=телефон_для_СБП  # Опционально
PORT=10000  # Опционально, по умолчанию 10000
```

**Как получить ADMIN_CHAT_ID:**
1. Откройте бота [@userinfobot](https://t.me/userinfobot) в Telegram
2. Отправьте ему команду `/start`
3. Он отправит вам ваш Chat ID (например: `123456789`)

### 4. Настройка реквизитов в коде

Откройте `bot.py` и замените плейсхолдеры:

- `[укажи свой адрес]` → реальный адрес студии
- `[укажи контакт]` → контакт администратора (телефон, Telegram и т.д.)

### 5. Запуск бота

```bash
python bot.py
```

## 📦 Деплой на Render

### 1. Создать Web Service на Render

1. Перейдите на [Render](https://render.com/) и зарегистрируйтесь
2. Нажмите **New +** → **Web Service**
3. Подключите свой GitHub-аккаунт
4. Выберите репозиторий с ботом
5. Заполните настройки:
   - **Name**: `machata-studio-bot`
   - **Region**: выберите ближайший регион
   - **Branch**: `main`
   - **Runtime**: `Python 3`
   - **Build Command**: `pip install -r requirements.txt`
   - **Start Command**: `python bot.py`

### 2. Добавить переменные окружения

В настройках сервиса перейдите в **Environment** и добавьте:

- `BOT_TOKEN` - ваш API Token от BotFather
- `ADMIN_CHAT_ID` - ваш Telegram Chat ID (опционально)
- `PAYMENT_CARD` - номер карты/кошелька для оплаты
- `PAYMENT_PHONE` - телефон для СБП (опционально)
- `PORT` - порт (по умолчанию 10000)

### 3. Запустить бота

1. Render автоматически запустит деплой
2. Дождитесь статуса **Live**
3. Откройте своего бота в Telegram и нажмите `/start`

## ⚙️ Настройка студий

Чтобы добавить несколько студий, откройте `bot.py` и измените словарь `STUDIOS`:

```python
STUDIOS = {
    'studio_main': {
        'name': 'Главная студия',
        'description': 'Репетиционная студия с полным оборудованием',
        'price_per_hour': 700
    },
    'studio_vip': {
        'name': 'VIP студия',
        'description': 'Премиум студия с профессиональным оборудованием',
        'price_per_hour': 1200
    }
}
```

## 💳 QR-код для оплаты

Бот автоматически генерирует QR-код для оплаты предоплаты (50% от суммы).

Для работы QR-кода установите переменные окружения:
- `PAYMENT_CARD` - номер карты/кошелька
- `PAYMENT_PHONE` - телефон для СБП (опционально, для формата СБП)

## 🔔 Уведомления администратора

Если установлена переменная `ADMIN_CHAT_ID`, администратор будет получать уведомления о:
- Новых бронированиях
- Данных пользователя
- Деталях бронирования

## 📝 Важные замечания

⚠️ **Безопасность:**
- Никогда не публикуйте токен бота в коде
- Всегда используйте переменные окружения
- Регулярно проверяйте логи

⚠️ **Хранилище данных:**
- Текущая версия использует хранилище в памяти
- При перезапуске бота все бронирования теряются
- Для продакшена рекомендуется использовать базу данных (SQLite, PostgreSQL и т.д.)

## 🛠️ Технические детали

- **Библиотека**: aiogram 3.x
- **Python**: 3.8+
- **FSM**: Используется для управления состояниями бронирования
