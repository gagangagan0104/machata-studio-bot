# Инструкция по настройке бота с ЮKassa через вебхуки

## ✅ Что изменено

Бот переведен на использование **API ЮKassa через вебхуки** вместо нативных методов Telegram Payments API (`sendInvoice`).

## 📋 Необходимые данные

1. **YOOKASSA_SHOP_ID** - Идентификатор вашего магазина (например: `390540012`)
2. **YOOKASSA_SECRET_KEY** - Секретный ключ от магазина (начинается с `live_` или `test_`)
3. **WEBHOOK_URL** - URL для вебхуков (например, через ngrok: `https://xxxx.ngrok.io`)

## 🔧 Настройка

### 1. Обновите `run_bot.command`

Откройте файл `run_bot.command` и укажите правильные значения:

```bash
export YOOKASSA_SHOP_ID="ваш_shop_id"
export YOOKASSA_SECRET_KEY="ваш_секретный_ключ"  # Полный ключ, например: live_xxxxxxxxxxxxx
export WEBHOOK_URL=""  # Оставьте пустым для локального запуска
```

### 2. Для локального тестирования (с ngrok)

1. Установите ngrok: https://ngrok.com/
2. Запустите бота: `./run_bot.command`
3. В другом терминале запустите ngrok:
   ```bash
   ngrok http 10000
   ```
4. Скопируйте HTTPS URL (например: `https://xxxx.ngrok.io`)
5. Обновите `run_bot.command`:
   ```bash
   export WEBHOOK_URL="https://xxxx.ngrok.io"
   ```
6. Перезапустите бота

### 3. Настройка вебхуков в личном кабинете ЮKassa

1. Войдите в личный кабинет ЮKassa: https://yookassa.ru/my
2. Перейдите в раздел **"Настройки" → "HTTP-уведомления"**
3. Добавьте URL вебхука: `https://ваш-домен.ru/payment` или `https://xxxx.ngrok.io/payment`
4. Включите события:
   - ✅ `payment.succeeded` (обязательно)
   - ✅ `payment.waiting_for_capture` (рекомендуется)

## 🚀 Запуск

1. Убедитесь, что все переменные окружения установлены в `run_bot.command`
2. Запустите бота: `./run_bot.command`
3. Бот будет работать в режиме polling (локально)

## 📝 Как это работает

1. Пользователь создает бронирование в боте
2. Бот создает платеж через API ЮKassa
3. Пользователю отправляется ссылка на оплату
4. После оплаты ЮKassa отправляет вебхук на `/payment`
5. Бот обновляет статус брони и уведомляет пользователя

## ⚠️ Важно

- **YOOKASSA_SECRET_KEY** должен быть полным ключом (например: `live_xxxxxxxxxxxxx`), а не просто `live_85738`
- Для продакшена используйте реальный домен, а не ngrok
- Убедитесь, что вебхук доступен по HTTPS
- Проверьте, что события `payment.succeeded` и `payment.waiting_for_capture` включены в ЛК ЮKassa

## 🔍 Проверка работы

1. Создайте тестовое бронирование в боте
2. Проверьте, что ссылка на оплату отправляется
3. Оплатите тестовой картой (для тестового режима: `1111 1111 1111 1026`)
4. Проверьте логи бота - должно быть сообщение о получении вебхука
5. Проверьте, что пользователь получил уведомление об успешной оплате

## 📞 Поддержка

Если возникли проблемы:
1. Проверьте логи бота
2. Проверьте настройки в личном кабинете ЮKassa
3. Убедитесь, что вебхук доступен и возвращает `200 OK`
